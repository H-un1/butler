# 버틀러(Butler) — Phase 분리 계획

> 풀스코프 실제 출시 제품이지만, 한 번에 다 만들면 품질이 떨어집니다.
> Phase별로 나눠 각각 "진짜 동작하는 제품"을 출시합니다.
> 의존성 순서: 대시보드/HouseLog → 점검 → 정산 → CRM → 고도화.

---

## Phase 1: 기반 + 비대면 핵심 (MVP)

### 목표
원격 임대인이 **가입 → 내 집 등록 → 공공데이터 대시보드 확인 → 점검 의뢰 → PDF 리포트 수령**까지 비대면으로 완주할 수 있다. House Log에 첫 데이터가 쌓이기 시작한다.

### 기능
- [ ] 회원/인증 — 임대인·임차인·점검자·관리자 역할, 소셜 로그인(카카오/네이버) + PASS 본인인증, RBAC
- [ ] 물건 등록 + 공공데이터 연동 (국토부 실거래가 / 건축물대장 / K-APT)
- [ ] 자산 대시보드 — 시세 트래킹, 단지정보, AMI 점수
- [ ] House Log — 생애주기 기록 원장 (append-only)
- [ ] 스마트 점검 — 점검자앱 체크리스트 → 사진 → AI 결함분석(보조) → PDF 리포트 자동생성 → 임대인 공유
- [ ] 구독 과금 — 보유 물건수 + 구간 할인 (구독 결제 포함)

### 데이터
- User, Property, Lease, HouseLogEntry, Inspection, InspectionItem, Report, Subscription

### 인증
- 소셜 로그인(OAuth: 카카오/네이버) + PASS 본인인증. 주민번호 비저장, JWT 세션, 역할 기반 권한(RBAC).

### "진짜 제품" 체크리스트
- [ ] 실제 DB 연결 (목업 데이터 X)
- [ ] 실제 인증 (하드코딩된 비밀번호 X)
- [ ] 실제 공공데이터 API 연동 (더미 응답 X)
- [ ] 실제 서버에 배포 (AWS, localhost X)
- [ ] 점검자 앱 / 임대인 앱 / 웹 관리자가 같은 서버에 붙어 동작
- [ ] 다른 사람이 앱을 설치해 실제 물건을 등록·점검 의뢰해볼 수 있음

## Phase 2: 차별화 무기 + 거래

### 전제 조건
- Phase 1이 안정적으로 배포되고, 점검 데이터(Inspection/Report)가 House Log에 쌓이는 상태.

### 목표
점검 데이터를 기반으로 **수선비를 객관적으로 정산**하고, 임대차 일상을 **CRM·자동알림**으로 관리하며, **앱 내 결제**까지 닫는다.

### 기능
- [ ] 수선비 정산 Settlement (룰베이스: LH 부담기준표 + 표준 내구연수 + 감가상각)
- [ ] 정산 리포트 + 임대인·임차인 양측 합의 플로우
- [ ] 임대차 CRM + 자동알림 (계약만료 D-Day / 월세미납 / 수선요청)
- [ ] 수선요청 이슈 협업보드 (임대인·임차인·관리자, 사진·진행상황·이력 아카이빙)
- [ ] PG 결제 (구독료 + 정산금 앱 내 결제)

### 추가 데이터
- Settlement, MaintenanceRequest, Payment, Notification

### 통합 테스트
- Phase 1 기능(대시보드/점검/House Log)이 여전히 정상 동작하는지 회귀 확인.
- 정산이 실제 점검 데이터(InspectionItem 등급·내구연수)를 정확히 참조하는지 검증.

### 주의사항
- **정산은 "산출+합의"까지만 자동화**하고, 보증금 자동 공제 같은 법적 리스크는 양측 합의 후 결제로 분리.
- PG 연동 시 정산금 분쟁·환불 책임 정책을 사전에 정의.

## Phase 3: 고도화 + 생태계

### 전제 조건
- Phase 1 + 2가 안정적으로 운영되고 사용자·데이터가 축적된 상태.

### 목표
정보 비대칭 해소(AI 상담), 커뮤니티 형성, 권리 리스크 진단, 보수업체 생태계로 플랫폼을 확장.

### 기능
- [ ] AI 챗봇 — 임대차법/세무 상담 (RAG 기반, 분쟁사례 가이드)
- [ ] 단지 커뮤니티 + 전자투표 — 실소유주 인증 폐쇄형, 단지 뉴스 피드
- [ ] AI OCR 등기부등본 — 권리관계 파싱 + 안전등급(깡통전세 진단), **주민번호 마스킹/비저장**
- [ ] 검증 보수업체 매칭 — 업체 디렉토리/평점 (필요 시 견적 입찰)
- [ ] 판례 기반 AI 보조 고도화 — 룰베이스 정산에 유사 분쟁사례 안내 강화

### 주의사항
- AI OCR의 개인정보(주민번호) 처리·위변조 탐지·법적 책임 범위 사전 법률 검토 필수.
- AI 챗봇·OCR은 외부 모델 API 비용 발생 → 과금 구조에 반영.
- 커뮤니티는 실소유주 인증 게이트와 신고·모더레이션 정책 필요.
