## 1. 제품 개요

### 한 줄 요약
버틀러는 **원격 임대인**이 멀리 있는 집을 비대면으로 관리하고, 주택의 전 생애주기(점검·수리·계약)를 **House Log**라는 디지털 원장에 기록해 자산가치를 증명하고 분쟁을 예방하는 **부동산 자산관리 SaaS 플랫폼**(모바일 앱 + 웹 관리자)이다.

### 해결하는 문제
1. **원격 관리 불가** — 거주지와 물건지가 다른 임대인이 늘었지만(갭투자·다주택), 물리적 거리 때문에 직접 현장 점검·대응이 불가능하다.
2. **관리 사각지대** — 누수·보일러 등 시설 민원에 즉각 대응이 어려워 세입자 불만과 임대인 스트레스가 누적된다.
3. **기준 없는 수선비 분쟁** — 표준 부담 기준이 없어 업체 견적 편차가 크고, 퇴거 정산 시 임대인-임차인 간 감정·법적 분쟁이 발생한다.
4. **자산 가치 증명 데이터 부재** — 수리·점검 이력이 휘발되어, 매매 시 "이 집이 잘 관리됐다"를 객관적으로 증명할 수 없다.
5. **프롭테크의 사후관리 공백** — 기존 플랫폼은 매물탐색·계약(일회성 중개)에만 집중하고, 계약 이후 2년 이상의 거주기간 관리 도구는 전무하다.

### 핵심 가치 (차별점)
- **House Log** — 주택 생애주기를 불변 타임라인으로 기록 → 자산가치 증명 + 분쟁 예방의 데이터 원장.
- **데이터 기반 수선비 정산** — LH 부담기준표 + 표준 내구연수 + 감가상각을 적용한 **룰베이스 정산 엔진**(AI는 유사 분쟁사례 안내 보조)으로, 설명 가능하고 법적으로 안전한 분담비율을 제시.
- **비대면 점검 대행 인프라** — **중개인**이 현장을 방문해 앱으로 체크리스트·사진을 기록하고 PDF 리포트를 자동 발행 → 원격 임대인도 "내 집 상태"를 받아본다.
- **구독형(SaaS) 전환** — 일회성 중개수수료가 아닌, 보유 물건수 기준 상시 관리 구독 모델.

> **포지셔닝 주의:** 버틀러는 "중개 플랫폼"이 아니라 "계약 이후 관리 플랫폼"이다. 매물탐색·중개는 범위 밖.

## 2. 사용자

이 서비스는 **다면 플랫폼**이다. 5개 역할이 존재한다.

### 핵심 사용자 (돈 내는 고객): 원격 임대인 (LANDLORD)
- **누구**: 거주지 ≠ 물건지인 다주택/갭투자 소유주. 비대면 관리 수요가 큼.
- **상황**: 멀리 있어 직접 갈 수 없는 집의 상태·계약·분쟁을 관리해야 할 때.
- **목표**: 내 집 상태를 원격으로 확인하고, 점검·수선·정산을 대행받고, 분쟁을 데이터로 예방.

### 임차인 (TENANT) — 전용 앱/계정
- **상황**: 거주 중 수선요청, 점검 리포트 확인, 퇴거 시 수선비 정산 합의.
- **목표**: 투명한 수선 처리와 공정한 보증금 정산.

### 점검자 / 현장 관리자 (INSPECTOR) — 중개인 점검 대행
- **상황**: 임대인 의뢰를 받아 **중개인**이 현장을 방문, 점검자 앱으로 체크리스트·사진 기록.
- **목표**: 표준화된 점검 수행과 품질 보증형 리포트 생성.

### 관리자 (ADMIN)
- **상황**: 웹 관리자에서 회원·물건·점검·정산·구독을 운영·검수.

### (Phase 3) 보수업체 (VENDOR)
- 검증된 업체로 등록되어 매칭·시공.

### 사용자 시나리오 (핵심)
1. **원격 임대인**이 가입(소셜 로그인 + PASS 본인인증)하고 본인 소유 **물건을 등록**한다.
2. 주소 입력만으로 **공공데이터(국토부 실거래가·건축물대장·K-APT)가 자동 연동**되어 시세·단지정보·AMI 점수가 대시보드에 표출된다.
3. 임대인이 **정기 점검을 의뢰**하면, **중개인**이 현장을 방문해 앱 체크리스트·사진을 기록한다.
4. 시스템이 **PDF 점검 리포트를 자동 생성**해 임대인 앱으로 도착 알림을 보내고, **House Log에 영구 저장**한다.
5. (Phase 2) 퇴거 시 점검 데이터 기반으로 **수선비 분담비율이 자동 산출**되고, 임대인·임차인이 앱에서 합의·결제한다.

| 자산 대시보드 + AMI | 실거래가 트래킹, 단지정보, 자체 자산관리지수(AMI) 점수화 | P1 (MVP) | 보통 |
| House Log | 주택 생애주기(점검·수리·계약·소유주변경) 불변 타임라인 기록 | P1 (MVP) | 보통 |

| 수선비 정산 (Settlement) | LH기준+감가상각 룰베이스 분담비율 산출 + 양측 합의 플로우 | P2 | 복잡 |

## 4. 사용자 흐름 (User Flow)

### 핵심 흐름 (Phase 1 — 원격 임대인 중심)
```
가입·본인인증 → 물건 등록(주소) → 공공데이터 자동연동
→ 대시보드 확인(시세·AMI) → 점검 의뢰 → [점검자 현장 방문·기록]
→ PDF 리포트 자동생성 → 임대인 알림 + House Log 저장
```

### 상세 흐름
1. **가입·인증**: 카카오/네이버 소셜 로그인 → PASS 본인인증(실소유주 확인). 주민번호는 저장하지 않고 인증 결과(verified_at)만 보관.
2. **물건 등록**: 도로명 주소 + 동/호 입력 → 시스템이 공공 API 호출하여 건축물대장·실거래가·K-APT 데이터 수집.
3. **대시보드**: 실거래 시세 트래킹, 준공연차·세대수·주차·관리비, AMI(자산관리지수) 점수 표출.
4. **점검 의뢰**: 임대인이 점검 유형(정기/수리전후/퇴거) 선택 후 일정 요청 → 점검자 배정.
5. **현장 점검(점검자 앱)**: 구역별 체크리스트 작성, 사진/동영상 업로드, 파손 부위 마킹, 항목별 등급(A~F).
6. **리포트 생성**: 업로드 이미지 AI 결함분석(보조) + 이전 점검 비교(Change Log) → 전문 템플릿 PDF 자동 병합.
7. **전송·아카이빙**: 임대인 앱 도착 알림(Push) + 임차인 공유(선택) + House Log DB 영구 저장. (카톡/이메일 전송은 Phase 2 알림과 함께)

## 5. 성공 기준

- [ ] 임대인이 주소 입력만으로 30초 내 공공데이터 연동 대시보드를 볼 수 있다.
- [ ] 점검자가 현장에서 앱으로 점검을 마치면 5분 내 PDF 리포트가 자동 생성된다.
- [ ] 생성된 모든 점검 리포트가 해당 물건의 House Log 타임라인에 누락 없이 기록된다.
- [ ] 임대인이 구독을 시작하고 보유 물건수에 따라 구간 할인이 정확히 과금된다.
- [ ] (Phase 2) 퇴거 정산 시 산출된 분담비율에 LH기준·감가상각 근거가 리포트로 첨부된다.
- [ ] 주민등록번호 등 고유식별정보가 DB에 평문 저장되지 않는다 (개인정보 컴플라이언스).

## 6. 안 만드는 것 (Out of Scope)

> 이 목록은 Phase 1에서 만들지 않습니다. AI에게 코드를 시킬 때 함께 공유하세요.

- **매물탐색·중개 기능** — 이유: 버틀러는 "계약 이후 관리" 플랫폼. 중개는 포지셔닝상 범위 밖.
- **수선비 정산 Settlement Logic** — 이유: 점검 데이터가 선행돼야 함 → Phase 2.
- **임대차 CRM·자동알림·이슈보드** — 이유: Phase 2.
- **PG 결제(정산금 결제)** — 이유: Phase 2. (단, 구독 과금 결제는 Phase 1에 필요)
- **AI 챗봇 / 단지 커뮤니티 / 전자투표** — 이유: Phase 3 고도화.
- **AI OCR 등기부등본 권리분석** — 이유: 개인정보·법적 검토 필요 → Phase 3.
- **보수업체 매칭** — 이유: 양면 업체 풀 확보 필요 → Phase 3.
- **판례 학습 AI 모델 직접 구축** — 이유: 정산은 룰베이스가 본체. 판례 AI는 보조이며 데이터·법적책임 리스크로 Phase 3 이후.

## 7. [NEEDS CLARIFICATION]

> 개발 전에 정해야 할 미결 사항.

- [ ] 공공데이터 API 실제 가용성: 국토부 실거래가/건축물대장/K-APT 각각의 신청 절차·요금·호출제한·실시간성 확인 필요.
- [ ] PASS 본인인증 연동 비용 및 실소유주(등기상 소유자)와 인증 명의 일치 검증 방법.
- [ ] 점검 수행 주체: **중개인**이 현장 점검 진행으로 확정 (검토했던 대안: 주택관리업체 / 시설관리업체 / 홈서비스 플랫폼(숨고 등)). 향후 중개인 제휴·교육·정산 정책 정의 필요.
- [ ] AMI(자산관리지수) 산출 공식의 구체적 가중치 정의 (노후도/세대수/주차/브랜드 등).
- [ ] 구독 과금 구간 정의: 채당 단가, 구간 경계(예: 1~3채/4~10채/11채+), 무료 체험 여부.
- [ ] House Log 데이터 소유권 정책: 임대인이 집을 매도하면 데이터는 누구에게 귀속·이전되는가.
- [ ] 점검 리포트의 법적 효력 범위: "양측 서명" 전자서명 도입 여부.
